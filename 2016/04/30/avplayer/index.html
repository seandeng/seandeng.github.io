AsyncDisplayKit 中的 ASVideoNode

发表于 2016-04-30   |     |   阅读次数

AsyncDisplayKit 对视频播放的支持很弱
它本身的ASVideoNode 是刚做出来的 这个视频播放器只有播放暂停的功能。

官方：

ASVideoNode is not yet optimized and should be used with caution for production purposes. Interested clients should follow developments in master, as performance in scrolling views with many ASVideoNodes will be improved in the next several weeks.

大意就是这个视频播放性能上还未被优化 在一个滚动条中有多个视频的方式将会在未来几周得到优化

自己动手丰衣足食

ASVideoNode 是对 AvPlayer 的封装。

AVPlayer : 播放器
AVPlayerItem : 播放器需要播放的资源,比如一首歌曲
CMTime : 记录AVPlayerItem资源的播放进度以及这个资源的其他信息

添加视频播放的进度条功能：

就是定时去刷新进度条UIProgressView

‘CMTimeGetSeconds(self.player.currentTime)’ 得到的现在播放时间
‘CMTimeGetSeconds(self.player.currentItem.duration)’ 是视频总时间, 相除就是进度条百分比了。

添加全屏播放功能：

一个小技巧就是 如果是在一个 tableview 的 cell 中放大旋转view至整个屏幕 是没办法成功的 因为父容器就那么大 不可能占满整个屏幕的
这个时候可以用 ‘[[UIApplication sharedApplication].keyWindow addSubnode:self];’ 的方式 把播放器放到keywindow上
缩小还原的时候就放回原来的父容器
所以整个播放器初始化的时候可以设置两个值，一个是在tableview 中的初始frame，一个是父容器对象，方便缩小的时候加回到原来的位置。
